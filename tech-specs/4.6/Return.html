Business Object: Return

Definition

Returns (Credit memo) are used to issue credit for sold Inventory

Properties:

Name Type Reference
number String that is the documentKey and idAttribute
returnDate Date required default today
isPosted Boolean required, defaulting to false, read only
isVoid Boolean required, defaulting to false, read only

taxTotal Money the sum of all taxes inluding line items, freight and tax adjustments
miscCharge Money read only
total Money the calculated total of subtotal + freight + tax + miscCharge
balance Money the sum of total - allocatedCredit - authorizedCredit - outstandingCredit. - If sum calculates to less than zero, then the balance is zero.

Related Information

    Privileges

  • Users can create, update, and delete Returns if they have the MaintainCreditMemos privilege.
  • Users can read Returns if they have the ViewCreditMemos privilege.

    Other

  • The Return collection is not cached.
  • Return is lockable.
  • The ID attribute is "number", which will be automatically uppercased.
  • Used in the billing module
  • Currency field should be read only after a line item is added to the Return
  • The Return numbering policy can be determined by the user.
  • When the customer changes, the billto information should be populated from the customer, along with the salesRep, commission, terms, taxZone, and currency. The billto fields will be read-only if the customer does not allow free-form billto.
  • When currency or Return date is changed outstanding credit should be recalculated.
  • AllocatedCredit should be recalculated when XM.ReturnAllocation records are added or removed.
  • When Return date is changed allocated credit should be recalculated.
  • When subtotal, totalTax or miscCharge are changed, the total should be recalculated.
  • TotalTax should be recalculated when taxZone changes or taxAdjustments are added or removed.
  • When an Return is loaded where "isPosted" is true, then the following attributes will be made read only: lineItems, number, returnDate, terms, salesrep, commission, taxZone, saleType
  • Balance should be recalculated when total, allocatedCredit, or outstandingCredit are changed.
  • When allocatedCredit or lineItems exist, currency should become read only.
  • To save, the Return total must not be less than zero and there must be at least one line item.
  • Return includes a function "calculateTax" that Gathers line item, freight and adjustments Groups by and sums and rounds to XT.MONEY_SCALE for each tax code Sums the sum of each tax code and sets totalTax to the result
  • When a customer with non-base currency is selected the following values should be displayed in the foreign currency along with the values in base currency - Unit price, Extended price, Allocated Credit, Authorized Credit, Margin, Subtotal, Misc. Charge, Freight, Total, Balance
  • A list view should exist called XV.ReturnList. Users can perform the following actions from the list: Delete unposted Returns where the user has the MaintainCreditMemos privilege, Post unposted Returns where the user has the "PostARDocuments" privilege, Void posted Returns where the user has the "VoidPostedARCreditMemos" privilege, Print Return forms where the user has the "PrintCreditMemos" privilege.
  • The Return list should not support multiple selections
  • The ReturnList should be printable
  • Supports grid-entry of line items on desktop browsers.
  • A Tax adjustments panel should be available. User shold be able to add new tax adjustments and remove tax adjustments for unposted Returns
  • A Credit Allocation panel should be available. When 'New' button is selected, user should be allowed to create a minimalized version of cash receipt on the fly
  • The cash receipt need only record the amount, currency, document number, document date, distribution date and whether the balance should generate a credit memo or a customer deposit, depending on global customer deposit metrics
  • When clicked, an "allocate" button should present a list of open receivables that are credits that can be associated with the Return
  • The 2 buttons above should only be enabled if the user has the "ApplyARMemos" privilege"
  • The bill to addresses available when searching addresses should filter on the addresses associated with the customer's account record by default.
  • The customer search list should search only on active customers.
  • A child workspace view should exist called XV.ReturnLineWorkspace should include: all the attributes on XM.ReturnLine, item cost and item list price values, and a read only panel that displays a group box of lists of taxes.
  • Return will include authorizedCredit, the sum of credit card authorizations in the order currency where: - The current_timestamp - authorization date is less than CCValidDays || 7 - The payment status the cc payment (ccpay) record is authorized ("A") - The cc payment record is for an order number = the order number specified on the Return When currency or Return date is changed authorized credit should be recalculated.
  • The project attribute will be read-only for posted Returns
  • The project widget will be added to the Return workspace if the UseProjects setting is true.
