Business Object: Invoice

Definition

Here is some high-level description of what an invoice is supposed to do.

Related Information

    Setup

  • Used in the billing module

    Privileges

  • Users can create, update, and delete invoices if they have the MaintainMiscInvoices privilege.
  • Users can read invoices if they have the ViewMiscInvoices privilege.

    Settings

  • Invoice is lockable.
  • The ID attribute is "number", which will be automatically uppercased.
  • There is a setting "Valid Credit Card Days"
    Default Value:
    • 7
  • Characteristics can be assigned as being for invoices
  • Currency field should be read only after a line item is added to the invoice
  • The invoice numbering policy can be determined by the user.
  • When the customer changes, the billto information should be populated from the customer, along with the salesRep, commission, terms, taxZone, and currency. The billto fields will be read-only if the customer does not allow free-form billto.
  • When currency or invoice date is changed outstanding credit should be recalculated.
  • AllocatedCredit should be recalculated when XM.InvoiceAllocation records are added or removed.
  • When invoice date is changed allocated credit should be recalculated.
  • When subtotal, totalTax or miscCharge are changed, the total should be recalculated.
  • TotalTax should be recalculated when taxZone changes or taxAdjustments are added or removed.
  • When an invoice is loaded where "isPosted" is true, then the following attributes will be made read only: lineItems, number, invoiceDate, terms, salesrep, commission, taxZone, saleType
  • Balance should be recalculated when total, allocatedCredit, or outstandingCredit are changed.
  • When allocatedCredit or lineItems exist, currency should become read only.
  • To save, the invoice total must not be less than zero and there must be at least one line item.
  • The invoice list should support multiple selections
  • Invoice will include authorizedCredit, the sum of credit card authorizations in the order currency where: - The current_timestamp - authorization date is less than CCValidDays || 7 - The payment status the cc payment (ccpay) record is authorized ("A") - The cc payment record is for an order number = the order number specified on the invoice When currency or invoice date is changed authorized credit should be recalculated.
  • sales extension order date defaults to today
  • The project attribute will be read-only for posted invoices
  • There is a setting "Valid Credit Card Days"
    Default Value:
    • 7

    Buttons

  • The InvoiceList should be printable
  • The InvoiceWorkspace should be printable
  • The InvoiceWorkspace should be printable

    Navigation

  • Users can perform the following actions from the list: Delete unposted invoices where the user has the MaintainMiscInvoices privilege, Post unposted invoices where the user has the "PostMiscInvoices" privilege, Void posted invoices where the user has the "VoidPostedInvoices" privilege, Print invoice forms where the user has the "PrintInvoices" privilege.
  • Supports grid-entry of line items on desktop browsers.
  • The bill to addresses available when searching addresses should filter on the addresses associated with the customer's account record by default.
  • The customer search list should search only on active customers.

    Other

  • The invoice collection is not cached.
  • Invoice includes a function "calculateTax" that Gathers line item, freight and adjustments Groups by and sums and rounds to XT.MONEY_SCALE for each tax code Sums the sum of each tax code and sets totalTax to the result
  • A child workspace view should exist called XV.InvoiceLineWorkspace should include: all the attributes on XM.InvoiceLine, item cost and item list price values, and a read only panel that displays a group box of lists of taxes.
  • The project widget will be added to the invoice workspace if the UseProjects setting is true.
