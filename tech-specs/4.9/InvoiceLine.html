Business Object: InvoiceLine

Definition

Represents a line of an invoice. Only ever used within the context of an invoice.

Related Information

    Privileges

  • User requires the "OverrideTax" privilege to edit the tax type.

    Settings

  • InvoiceLine keeps track of the available selling units of measure based on the selected item, in the "sellingUnits" property
  • Quantity and billed values can be fractional only if the item allows it
  • The "ordered" and "billed" amounts must be positive
  • When item is unset, all item-related values should be cleared.
  • The user can define a line item as being miscellaneous or not. Miscellaneous means that they can enter a free-form itemNumber, itemDescription, and salesCategory. If the item is not miscellaneous then they must choose an item instead.
  • The price will be recalculated when the units change.
  • If price or billing change, extendedPrice should be recalculated.
  • When billed is changed extendedPrice should be recalculated.

    Other

  • When the item is changed the following should be updated from item information: sellingUnits, quantityUnit, quantityUnitRatio, priceUnit, priceUnitRatio, unitCost and taxType. Then, the price should be recalculated.

    InvoiceLineTax

  • Contains the tax of an invoice line.
