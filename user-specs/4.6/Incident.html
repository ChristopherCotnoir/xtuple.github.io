Business Object: Incident

Definition

The Incident Management system enables you to monitor and record Incidents that are linked to Accounts, Contacts, Users, and To-Do"s.

Related Information

    Setup

  • Used in the crm module

    Privileges

  • Incidents can be read by users with "ViewAllIncidents" privilege and can be created and updated by users with the "MaintainAllIncidents" privilege.
  • Incidents cannot be deleted by the users
  • Users with "ViewPersonalIncidents" privilege can read their personal Incidents but cannot read the contact's owned by other users. Users with "MaintainPersonalIncidents" privilege can create and update their personal Incidents but not the Incidents owned by other users

    Settings

  • Incidents are lockable.

    Navigation

  • Contacts search should display only contacts related to the specific account, if account is already selected

    Other

  • The Incident has no cache defined
  • Comments panel should exist to add new comments to the Incident
  • All Comment Types assigned to Incident should be available for selection
  • Documents panel should exist to connect the Incidents to : Account, contact, Customer, File, Incident, Item, Link, Incident, Project, To do
  • TO DO panel should exist to attach existing/new To Dos to the Incident
  • User should be able to detach an attached To do from the Incident screen
  • User should be able to open an attached To Do from the Incident screen
  • New/Attach buttons in the 'TO DO' panel should be disabled for a new Incident
  • History panel should be available which displays the Incident history
  • Characteristics Drop down list should display the characteristics assigned to the Incidents
