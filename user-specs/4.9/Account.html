Business Object: Account

Definition

Accounts are central to the xTuple CRM system. An Account holds information about an entity and serves to define it further as a Customer, Prospect, Vendor, Competitor, or Partner..

Related Information

    Other

  • The ID attribute is "number", which will not be automatically uppercased.
  • Used in the crm module
  • Accounts are lockable.
  • Account has no cache defined
  • Accounts can be read by users with "ViewAllCRMAccounts" privilege and can be created, updated, or deleted by users with the "MaintainAllCRMAccounts" privilege.
  • Users with "ViewPersonalCRMAccounts" privilege can read their personal accounts but cannot read the accounts owned by other users. Users with "MaintainPersonalCRMAccounts" privilege can create, update or delete their personal accounts but not the accounts owned by other users
  • Assigning an account to itself as its parent should display an error dialog 'Record is not allowed to reference itself as its parent'
  • Selecting one of Roles: Customer,Prospect,Employee,Sales Rep,Tax Authority, 'User account - should display the respective workspace with Number and Name populated by default
  • If in unsaved state, Selecting one of Roles: Customer,Prospect,Employee, Sales Rep,Tax Authority,User account - should display the respective workspace with Number and Name populated by default
  • Comments panel should exist to add new comments to the Account
  • Documents panel should exist to connect the Accounts to : Account, contact, Customer, File, Incident, Item, Link, Opportunity, Project, To do
  • Contacts panel should exist to attach existing/new contacts to the Account
  • TO DO panel should exist to attach existing/new To Dos to the Account
  • Opportunities panel should exist to attach existing/new Opportunities to the Account
  • Incidents panel should exist to attach existing/new Incidents to the Account
  • Projects panel should exist to attach existing/new Projects to the Account
  • Delete option should be disabled for the Accounts with a role
  • Delete option should be disabled for the Accounts assigned with a contact
  • Account Number should follow the Number policy in the Configure CRM screen
  • - Account Number should be blank for a new Account when Number policy is set to 'Manual'
  • - Account Number should be populated with 'Next Number' for a new Account when Number policy is set to 'Automatic'
  • - Account Number should be populated with 'Next Number' but editable for a new Account when Number policy is set to 'Override Allowed'
