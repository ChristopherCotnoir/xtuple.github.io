Business Object: ItemSource

Definition

Item Sources identify the relationship between Items and the Vendors who supply them

Properties:

Name Reference
id
uuid
item Enter the Item Number of the Purchase Item you want to create a Purchase Order for.
vendor Enter the Vendor Number of the Vendor who is the source for the specified Item.
isActive Identifies whether the Item is currently active.
isDefault Select this option to distinguish an Item Source from other Item Sources for the same Item Number. If selected, the default Item Source will be selected automatically when releasing Purchase Requests or creating drop shipments. If no default Item Source is specified, users will be asked to select Item Sources from a list of available Item Sources when releasing Purchase Requests or creating drop shipments. When this option is selected and saved, any other Item Sources for the Item flagged as the default will be unchecked.
vendorItemNumber Enter the Item Number the Vendor uses to identify the Item.
vendorItemDescription Enter the item number description the vendor uses to identify the basic type of item
vendorUnit Enter the Unit of Measure the Vendor uses for the specified Item.
vendorUnitRatio Specify the ratio between Inventory and Vendor Units of Measure UOM. For example, if you store an Item in single units, but a Vendor sells you the Item in cases of 12, then you would enter '12' here to represent the 12:1 Inv./Vend. ratio. If the ratio is 1:1, then enter '1' here. When an Inv./Vend. ratio is specified, the system allows you to purchase Items using the Vendor's UOM; however, when the Items are received, the system will automatically convert the Inventory to your Inventory UOM. This same conversion logic flows throughout the systemâ€”whether you are viewing Inventory availability or running Material Requirements Planning 'MRP'. Your Inventory will always be represented in your Inventory UOM, regardless of the Vendor UOMs specified on any open Purchase Orders.
notes This is a scrolling text field with word-wrapping for entering Notes related to the Item Source.
minimumOrderQuantity Indicate the minimum Order allowed for the specified Item. A warning message will be shown if a Purchase Order Line Item falls below the minimum Order value entered here.
multipleOrderQuantity Value entered indicates the quantity multiple that Orders for this Item must use. A warning message will be shown if a Purchase Order Line Item does not conform to the Order multiple entered here.
leadTime default to '0' Specify the lead time for the Vendor Item using the arrow buttons, or manually enter a value up to a maximum of 99. Specifies the Lead Time the Vendor requires to supply the specified Item, measured in days.
earliestDate Possible date of delivery for the ordered item
ranking Specify the Vendor's rank using the arrow buttons, or manually enter a value up to a maximum of 9. The ranking of a Vendor is used to identify the relative merits of Item Sources when releasing Purchase Requests. If a purchased Item has multiple Item Sources, you may select from a list of available Item Sources when releasing a Purchase Request. The Item Sources will be listed in descending order by Vendor ranking.
manufacturerName Name of the Manufacturer who creates the Item.
manufacturerItemNumber Item Number used by Manufacturer who creates the Item.
manufacturerItemDescription Description used by the Manufacturer who creates the Item.
barcode Enter the Bar Code for the Item. Note: Standard bar code formats supported by the system include the following: Code 3of9, Code 3of9 Extended,Code 128, UPC-A, UPC-E, EAN-8, EAN-13.
effective ItemSource prices will be effective from the given date.
expires ItemSource prices will be expired after the given date.
prices Display lists Item Source Prices for the specified Item Source.

Related Information

    Setup

  • Used in purchasing module

    Privileges

  • Users can create, update, and delete ItemSource if they have the 'MaintainItemSources' privilege.

    Settings

  • ItemSource is lockable.
  • The ID attribute is "uuid"
  • Marking 'Default' in the item source should use the item source in creating purchase order for the selected item-vendor combination
  • Unchecking 'Active' checkbox in the 'ItemSource' screen should de-activate the selected item source

    Buttons

  • Vendor panel should exist to display the item information linked to the vendor
  • Prices panel should exist to display pricing information of the item for selected vendor
  • 'New' button in the 'Prices' panel should be in de-activated state if no vendor is selected in the ItemSource screen
  • 'Delete','<','>' and 'Done' buttons should be in disabled mode if no price is selected in the 'Prices' panel of the selected item source

    Navigation

  • An Action gear exists in the 'ItemSources' work space with following options: 'Delete' option where the user has MaintainItemSources privilege
  • An Action gear exists in the 'ItemSources' work space with no 'Delete' option if the selected item source has purchase order linked to it

    Other

  • The Item Source Collection is not cached.
  • Negative values should not be allowed in 'Quantity Break' field
  • User should not allow to enter negative values in the 'Price' field of 'Prices' panel
  • Currency should be displayed in the 'Prices' panel according to the 'Vendor' selected in 'OverView' panel of the ItemSource
  • 'Site' drop down in the Prices panel should display the available sites in the database
  • 'Price Type' drop down of the 'Prices' panel should display the available Price Types
  • 'Price' drop down of the 'Prices' panel should display the available Currencies in the database and selecting non-base currency should convert the price to bases currency according to the Exchange specified
