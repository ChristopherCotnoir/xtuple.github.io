    Buttons

  • The SalesOrderWorkspace should be printable

    Other

  • Sales orders are lockable.
  • The ID attribute is "number", which will be automatically uppercased.
  • Used in the sales module
  • Sales Orders can be read by people with "ViewSalesOrders" and can be created, updated, or deleted by users with the "MaintainSalesOrders" privilege.
  • An Action gear exists in the Sales order workspace with two options - Issue to Shipping - Express Checkout
  • Selecting 'Issue to Shipping' opens the Issue to Shipping screen with the sales order number prepopulated
  • User will be asked to save the sales order when 'Issue to Shipping' action is selected if the sales order is in unsaved state
  • Express Checkout should do the following: - Issue all outstanding materials - Prompt user for distribution detail if applicable - Approve for billing - Create and print invoice
  • Express Checkout option will be only available to the user if 'IssueStockToShipping' and 'PostMiscInvoices' privileges are assigned to it
  • Characteristics can be assigned as being for sales orders
  • Workflows can be added, edited and removed from a new sales order
  • Workflows can be added, edited and removed from an existing sales order
  • Sales orders cannot be saved with incomplete workflows
  • When the sale type changes, the default hold type of the sale type will get copied to the sales order.
  • When the sale type changes, the characteristics of the new sale type are copied over to the sales order.
  • When the sale type changes, the workflow sources of the new sale type are transformed into workflow items for the sales order.
  • When a workflow item is completed or deferred, the hold type of the sales order will be set to be the applicable target hold type of the workflow item.
  • For the Workflow items copied from the Sale types, the start date and due date " + "should be calculated correctly based on the offset
  • When a Sale type with workflows, of an existing sales order is changed," + "to a sale type with no workflows, the existing workflows should be cleared" + "on the sales order
  • The due date for "Pack" workflow items will default to the "Pack date" on the order. Changing the Pack Date will update "Pack" workflow item's due date
  • The due date for "Ship" workflow items will default to the schedule date on the header. If that date changes, "Ship" workflow items will be updated.
  • When hold type of an order is changed to "None", all credit check type workflow items will be marked completed.
  • Selecting to change the sales order line item quantity should display a confirmation dialog asking whether to update the price or not.Selecting 'Yes', should update the price, selecting 'No' should remain the prices unchanged
